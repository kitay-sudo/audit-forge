Yata превращает любого кодового агента в дотошного аудитора безопасности. Дай агенту одну ссылку — он поймёт стек, проведёт глубокий аудит и создаст папку с отчётом и готовыми diff-фиксами.

Назван в честь Ята-но-Кагами (八咫鏡) — священного зеркала из японской мифологии, являющего истину.

Запусти прямо отсюда

Открой свой проект в AI-агенте (Claude Code, Cursor, Codex…) и вставь этот промпт. Копировать файлы в проект не нужно — агент сам всё скачает и выполнит.

your-project — agent prompt
# вставь это агенту внутри проекта, который проверяем
$ Скачай https://raw.githubusercontent.com/kitay-sudo/yata/main/YATA.md и выполни его целиком на ЭТОМ репозитории. Создай в корне проекта папку yata/ и сложи туда полный отчёт и по одному применяемому .diff-патчу на каждую уязвимость. Остальной репозиторий — только чтение; патчи не применяй, пока я не скажу.

Как это работает

Ссылка → аудит → папка с фиксами

Три шага. Никакой настройки, никакой установки, никакого вендор-лока.

01

Даёшь ссылку

Вставляешь агенту одну raw-ссылку на плейбук. Агент скачивает его и читает как инструкцию: что за проект, какой стек, как и в каком порядке проверять.

→
02

Агент проводит аудит

17 фаз по порядку: от разведки и threat-model до инъекций, БД, AI/LLM и rate-limit. Каждая находка — с реальным путём эксплуатации, без «воды» и ложных срабатываний.

→
03

Получаешь yata/

Самодостаточная папка: отчёт с вердиктом go/no-go, находки по severity и применяемые git apply diff-патчи. Остальной код агент не трогает.

Что внутри

Глубина уровня senior AppSec-инженера

Не просто чеклист — операционное руководство, по которому агент работает как опытный пентестер.

🧭

Понимает любой проект сам

Автоопределение стека по файлам-маркерам (Node, Python, Go, Java, Ruby, PHP, .NET, Rust, Docker/K8s/Terraform), карта точек входа, границ доверия и «crown jewels» — прежде чем что-то трогать.

🩹

Фиксы как diff, а не советы

Каждая уязвимость закрывается минимальным unified-diff патчем, который применяется через git apply. Идиоматично коду, без поломки логики.

🤖

AI / LLM безопасность

OWASP LLM Top 10 (2025): prompt injection, утечка system-prompt, excessive agency, RAG/vector, cost-DoS.

🔗

Supply chain 2026

CVE, SBOM (CycloneDX/SPDX), SLSA-provenance, подпись артефактов, пиннинг по SHA.

🚦

Rate limit и нагрузка

Где нужны лимиты, защита от исчерпания ресурсов, таймауты — чтобы не упасть под нагрузкой.

🎯

Дисциплина против false-positive

Каждая находка обязана иметь реальный достижимый путь эксплуатации и точную локацию file:line. Никаких «выглядит небезопасно».

✅

Вердикт «можно ли в прод»

Финальный production-readiness gate: чёткое go / go-with-conditions / no-go по 18 пунктам — секреты, авторизация, инъекции, лимиты, шифрование, логирование, инфра, AI и приватность. Чтобы деплоить без страха быть взломанным или словить утечку.

Результат

Одна папка yata/ — всё в ней

Самодостаточно и готово к ревью. Остальной репозиторий агент не модифицирует.

yata/ ├── README.md ← вердикт + severity + ссылки ├── SECURITY_AUDIT_REPORT.md ← полный отчёт ├── findings/ │ ├── CRIT-01-idor.md │ └── HIGH-01-sqli.md ├── fixes/ │ ├── CRIT-01-idor.diff git apply ✔ │ └── HIGH-01-sqli.diff ├── route-inventory.md └── appendix/ ← сырой вывод SCA/SAST

Индекс и отчёт

Вердикт go/no-go, счётчик находок по severity, таблица «находка → дифф» и полный отчёт по шаблону.

Находки по severity

Каждая с локацией, путём эксплуатации, CVSS (для High/Critical) и маппингом на OWASP/CWE.

Применяемые патчи

По одному .diff на проблему с единым ID (CRIT-01, HIGH-01…). Просмотри и применяй git apply.

⚠️ Только для авторизованного аудита

Yata предназначен для защитной проверки кода, который вам разрешено аудитировать: свой проект, проект работодателя, клиент с договором, CTF или исследование с разрешением. По умолчанию — статический/локальный анализ; любые динамические тесты только против сред, которыми вы владеете. Найденные секреты редактируются, а не выводятся. Это чеклист и навигатор для агента, а не гарантия — результат служит экспертным входом для решения человека.
